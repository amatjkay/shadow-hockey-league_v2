## 🗂️ Детализация этапов

### Этап 9.1: Миграция на `datetime.now(datetime.UTC)` (2 SP)

**Проблема:**
- 63 deprecation warnings в Python 3.14
- `datetime.utcnow()` устарел, заменён на `datetime.now(datetime.UTC)`

**Файлы для изменения:**
- `services/api_auth.py` — 2 места
- `models.py` — 1 место
- `tests/test_api.py` — 9 мест
- `tests/test_api_auth.py` — 5 мест
- `tests/test_audit_service.py` — 4 места
- `services/audit_service.py` — 1 место

**Критерий приёмки:** 0 deprecation warnings, все 239 тестов проходят

---

### Этап 9.2: Очистка старых audit_logs (1 SP)

**Проблема:**
- Таблица audit_logs будет расти со временем
- Требуется автоматическая очистка записей старше 90 дней

**Решение:**
- Использовать существующую функцию `cleanup_old_audit_logs(days_to_keep=90)`
- Настроить периодический вызов через:
  - OS cron (Linux) / Task Scheduler (Windows)
  - Или Celery Beat / APScheduler внутри приложения

**Критерий приёмки:** Автоматическая очистка работает, логи не растут бесконечно

### Этап 9.3: E2E тестирование production (3 SP)

**Проблема:**
- Локальные тесты проходят, но production может отличаться
- Требуется проверка на реальном сервере https://shadow-hockey-league.ru/

**Тест-кейсы:**
1. GET / — leaderboard загружается
2. GET /health — статус healthy
3. GET /metrics — Prometheus метрики
4. POST /admin/login — вход работает
5. CRUD через админку — данные создаются/обновляются/удаляются
6. Flush Cache — кэш сбрасывается
7. Статические файлы — CSS/JS/flags загружаются

**Критерий приёмки:** Все 7 кейсов проходят на production

---

### Этап 9.4: Мониторинг RAM (1GB VPS) (2 SP)

**Проблема:**
- VPS имеет 1GB RAM, Redis лимит 128MB
- Требуется мониторинг использования памяти

**Решение:**
- Настроить alerts в ztv.su при RAM > 80%
- Мониторить Redis memory usage через `/health`
- Добавить метрику `process_resident_memory_bytes` в Prometheus

**Критерий приёмки:** Alerts настроены, метрики собираются

---

### Этап 9.5: План миграции на PostgreSQL (4 SP)

**Проблема:**
- SQLite не масштабируется > 1000 менеджеров
- Требуется план миграции на PostgreSQL

**Решение:**
- Документация: пошаговый план миграции
- Тесты совместимости: проверить все запросы на PostgreSQL
- Alembic: настроить миграции для PostgreSQL
- CI/CD: добавить PostgreSQL в тестовое окружение

**Критерий приёмки:** Документация готова, тесты проходят на PostgreSQL
