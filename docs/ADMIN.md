## 🎯 Обзор

Админ-панель для управления данными лиги: страны, менеджеры, достижения, пользователи, API-ключи.

**URL:** `https://shadow-hockey-league.ru/admin/`
**Доступ:** Требуется аутентификация

## 🔐 Вход в систему

### Первый вход

1. **Создайте админа:**

   ```bash
   cd /home/shleague/shadow-hockey-league_v2
   source venv/bin/activate
   python scripts/create_admin.py
   ```

2. **Введите данные:**
   - Username: `admin` (или любой)
   - Password: минимум 6 символов

3. **Войдите:** `https://shadow-hockey-league.ru/admin/`

## 🛡 CSRF защита

Админ-панель защищена **CSRF (Cross-Site Request Forgery)** токенами через Flask-WTF.

### Как это работает

1. При загрузке формы генерируется уникальный CSRF-токен
2. При отправке формы токен проверяется на сервере
3. Запросы без валидного токена отклоняются (`400 Bad Request`)

### Конфигурация

CSRF защита включается переменной окружения:

```env
WTF_CSRF_SECRET_KEY=your-secret-key-here
```

> **Важно:** Используйте сложный секретный ключ в production (минимум 32 символа).

### Генерация секретного ключа

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

### Ограничения

- CSRF защита применяется ко всем POST/PUT/DELETE запросам через админ-панель
- API-эндпоинты используют API Keys auth (не CSRF)
- GET-запросы не требуют CSRF-токен

## 📊 Управление данными

### Страны (Countries)

**Поля:**

- `id` — уникальный идентификатор
- `code` — 3-буквенный код (RUS, BEL, USA)
- `name` — название страны (Russia, Belarus)
- `flag_path` — путь к флагу (выбирается из списка)

**Операции:**

- ✅ Создание / Редактирование / Удаление
- ✅ Поиск по коду и названию
- ✅ Фильтр по коду/названию

---

### Менеджеры (Managers)

**Поля:**

- `id` — уникальный идентификатор
- `name` — имя (уникальное)
- `country` — страна (выбор из списка)

**Тандемы:**

- Формат: `Vlad, whiplash 92` (через запятую)
- Отображаются с бейджем «тандем»

---

### Достижения (Achievements)

**Поля:**

- `id` — уникальный идентификатор
- `type` — тип (TOP1, TOP2, TOP3, BEST, R3, R1)
- `league` — лига (1 или 2)
- `season` — сезон (24/25, 23/24)
- `title` — название
- `icon_path` — путь к иконке
- `manager` — менеджер (выбор из списка)

**Фильтры:**

- По лиге
- По сезону
- По типу

---

### Пользователи (Admin Users)

**Поля:**

- `id` — уникальный идентификатор
- `username` — логин (уникальный)
- `password` — пароль (хешируется)

**Ограничения:**

- ❌ Нельзя удалить последнего админа

---

### 🔑 API-ключи (ApiKeys)

Управление ключами для доступа к REST API.

**Поля:**

| Поле           | Тип              | Описание                                           |
| -------------- | ---------------- | -------------------------------------------------- |
| `id`           | int              | Автоинкремент                                      |
| `name`         | str              | Человекочитаемое имя (e.g., "Integration Service") |
| `scope`        | str              | Уровень доступа: `read`, `write`, `admin`          |
| `key_hash`     | str              | SHA-256 хеш (отображается частично)                |
| `created_at`   | datetime         | Дата создания                                      |
| `last_used_at` | datetime         | Последнее использование                            |
| `expires_at`   | datetime \| None | Дата истечения                                     |
| `revoked`      | bool             | Флаг отзыва (True/False)                           |

#### Создание API-ключа

1. Перейдите в `/admin/apikey/`
2. Нажмите **«+ Add»**
3. Заполните поля:
   - **Name:** Описательное имя (e.g., "Mobile App", "CI/CD Pipeline")
   - **Scope:** Выберите уровень доступа
     - `read` — только GET запросы
     - `write` — GET, POST, PUT (без удаления)
     - `admin` — полный доступ (включая DELETE)
   - **Expires At:** (опционально) Дата истечения срока
4. Нажмите **«Create»**
5. **⚠️ Сохраните ключ немедленно!** — Он показывается только один раз в plain text.

> **Рекомендация:** Создавайте ключи с минимально необходимым scope. Используйте `read` для мониторинга, `write` для интеграций, `admin` только для доверенных сервисов.

#### Отзыв ключа

1. Найдите ключ в списке `/admin/apikey/`
2. Откройте запись для редактирования
3. Установите флаг **Revoked = True**
4. Нажмите **«Save»**

> **Внимание:** Отозванный ключ немедленно перестаёт работать. Восстановление невозможно — создайте новый.

#### Мониторинг использования

- **Last Used At:** Показывает последнее использование ключа
- Если ключ не использовался долгое время — рассмотрите отзыв
- Ключи с истёкшим `expires_at` автоматически отклоняются

#### Рекомендуемые практики

| Сценарий             | Scope   | Срок      | Примечание            |
| -------------------- | ------- | --------- | --------------------- |
| Мобильное приложение | `read`  | 365 дней  | Только чтение данных  |
| CI/CD интеграция     | `write` | 90 дней   | Обновление достижений |
| Админ-инструмент     | `admin` | 30 дней   | Временный доступ      |
| Мониторинг           | `read`  | Без срока | Health check, метрики |

## 🔄 Инвалидация кэша

Кэш главной страницы (leaderboard) автоматически сбрасывается при:

### Через админ-панель

- Создании/изменении/удалении менеджера
- Создании/изменении/удалении достижения
- Создании/изменении/удалении страны

### Через REST API

- Все CRUD операции с countries, managers, achievements

**TTL кэша:** 300 секунд (5 минут)
**Механизм:** `invalidate_leaderboard_cache()` вызывается в `on_model_change`/`on_model_delete` (admin) и после каждого commit (API)

## 🛡 Безопасность

### CSRF защита

- Все формы защищены CSRF-токенами (Flask-WTF)
- Требуется `WTF_CSRF_SECRET_KEY` в `.env`
- POST/PUT/DELETE запросы без токена отклоняются

### Требования к паролю

- Минимум 6 символов
- Рекомендуется: буквы + цифры + спецсимволы

### Аутентификация API

- API использует ключи с 3 уровнями доступа (read/write/admin)
- Ключи хешируются (SHA-256) перед хранением в БД
- Rate limiting: 100 req/min на ключ

### Аудит действий

Все изменения через админ-панель логируются Flask-Admin:

- Кто внёс изменение (username)
- Когда (timestamp)
- Что изменено (модель, ID, поля)

### Защита сессий

- Сессии используют `SECRET_KEY`
- Пароли хешируются (PBKDF2-SHA256)
- Сессии истекают при закрытии браузера
