## 1. Critical Deployment Fixes

### ❌ Problem: Windows Paths in `.env` after Deploy

- **Symptoms**: `OperationalError: no such table: managers`.
- **Cause**: `.env` on VPS contains a Windows path like `sqlite:///c:/dev/...`.
- **Solution**:
  ```bash
  # Connect to VPS and edit .env
  nano /home/shleague/shadow-hockey-league_v2/.env
  # Change DATABASE_URL to:
  DATABASE_URL=sqlite:////home/shleague/shadow-hockey-league_v2/instance/dev.db
  ```

### ❌ Problem: `systemctl` requires password

- **Solution**: Configure passwordless sudo for `systemctl` by creating `/etc/sudoers.d/shleague-systemctl`.

## 2. Database Issues

### ❌ Error: "no such table: managers"

- **Action**:
  1. Check if DB file exists: `ls -la instance/dev.db`.
  2. Run migrations: `alembic upgrade head`.
  3. Verify `.env` path (see Section 1).

### ❌ Error: "Database is locked"

- **Solution**: Enable WAL mode:
  ```bash
  sqlite3 instance/dev.db "PRAGMA journal_mode=WAL;"
  ```

## 3. Application Errors

### ❌ 500 Internal Server Error

- **Action**:
  ```bash
  journalctl -u shadow-hockey-league -n 50 --no-pager
  systemctl restart shadow-hockey-league
  ```

### ❌ 502 Bad Gateway

- **Action**: Check Gunicorn status and Nginx config.
  ```bash
  systemctl status shadow-hockey-league
  systemctl restart nginx
  ```

## 4. Bug Analysis (Known Issues)

### 🐛 BUG-08: API Returning Null for Achievement Relations

- **Status**: Under Investigation
- **Symptom**: `/admin/api/managers/{id}/achievements` returns empty strings for `type`, `league`, and `season`.
- **Root Cause**: Likely an issue with the SQLAlchemy loader strategy in `blueprints/admin_api.py`.
- **Temporary Fix**: Use explicit `db.session.get()` for related models in the API loop.

## 5. Infrastructure (Redis/Nginx)

### ❌ Redis Disconnected

- **Symptom**: Health check shows `redis_status: disconnected`.
- **Action**: `sudo systemctl restart redis-server`.

### ❌ CSS/JS 404

- **Action**: Check file permissions for the `static/` folder.
  ```bash
  chmod -R 755 static/
  ```
