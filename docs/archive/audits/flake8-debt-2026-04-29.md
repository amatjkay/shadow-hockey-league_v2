## 3. Action plan

The 9 hidden codes split into three buckets by safety / value:

### High-value real bugs (do first)
- **`F821` √ó 2** in `tests/integration/test_cache_invalidation.py:84-86` ‚Äî
  references to undefined `leagues` / `seasons` from a half-applied
  refactor. The test currently passes only because it never reaches that
  branch. **Fix imports or remove dead code path.**

### Code-quality cleanup (do next)
- **`F841` √ó 4** ‚Äî drop the unused locals.
- **`E711` √ó 2 / E712 √ó 1** ‚Äî replace `== None` / `== True` with `is`.
- **`E741` √ó 3** ‚Äî rename `l` ‚Üí something descriptive.

### Cosmetic (do last, ideally one mass commit)
- **`W503` √ó 9** ‚Äî disable globally in `.flake8` (PEP 8 itself
  acknowledges that "line break before binary operator" is now the
  preferred style).
- **`F541` √ó 7** (3 disappear after PR #41) ‚Äî convert `f"..."` with no
  placeholders back to plain strings.
- **`E402` √ó 5** ‚Äî `wsgi.py:15` is justified (sys.path manipulation
  before imports), so add a `# noqa: E402` there. The remaining 4 in
  scripts/ can be reordered.

## 4. Suggested cleanup PRs

1. **Real bugs** (`F821`) ‚Äî single small PR, high signal.
2. **Code-quality** (`F841`, `E711`, `E712`, `E741`) ‚Äî single PR, no
   behaviour change.
3. **Cosmetic** (`W503`, `F541`, `E402`) ‚Äî single PR + drop the
   suppressed codes from `.flake8 extend-ignore`.

After PR 3 lands, `.flake8 extend-ignore` should contain only `E203`
and `W503` (both are PEP 8-acknowledged style preferences rather than
real lint), and every other code in the current ignore list will be
caught by `make check` going forward.
