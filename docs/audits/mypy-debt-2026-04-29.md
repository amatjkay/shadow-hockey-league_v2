## 3. Suggested cleanup ordering

To avoid touching too many call sites at once, drive the cleanup
**from leaves toward the root**:

1. Start with the test files (49 of 92 errors live there). They never
   ship to prod, so changes carry zero runtime risk.
2. Move on to `services/*.py` next (22 errors); these are all
   well-defined helpers with stable signatures.
3. Tackle the remaining leaf modules — `data/seed_service.py` (4) and
   `blueprints/main.py` (3) — for 7 more errors with limited blast
   radius.
4. Finish with `models.py` and `app.py` (14 errors); these are the
   highest-risk because the SQLAlchemy 2.0 migration touches every
   query in the project.
