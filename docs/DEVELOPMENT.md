# Development, Testing & Deployment Guide

## 1. Development (Contributing)
### Code Style
- Follow PEP 8 for Python code.
- Use meaningful variable and function names.
- Keep functions small and focused on a single task.

### Workflow
1. Create a new branch for each feature or bugfix.
2. Write tests for your changes.
3. Ensure all tests pass before submitting a PR.
4. Update documentation if necessary.

---

## 2. Testing
### Running All Tests
```bash
pytest
```

### With Coverage Report
```bash
pytest --cov=. --cov-report=term-missing
```

### Test Structure
- `tests/unit/`: Logic, calculation, and model tests.
- `tests/integration/`: API, routes, and database constraints.
- `tests/e2e/`: End-to-end browser-based testing.

## 3. Deployment (Migration Guide)
### Server Requirements
- **OS**: Ubuntu 22.04 LTS
- **Python**: 3.10+
- **Redis**: 6.0+

### Step-by-Step Setup
1. **Clone Repository**: `git clone <url>`
2. **Virtual Environment**: `python3 -m venv venv && source venv/bin/activate`
3. **Dependencies**: `pip install -r requirements.txt`
4. **Environment**: Create `.env` based on `.env.example`.
5. **Database**: `alembic upgrade head`
6. **Services**: Configure Gunicorn and Nginx (see `docs/OPERATIONS.md` for details).
