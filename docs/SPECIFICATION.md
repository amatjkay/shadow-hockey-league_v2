## Обзор

**Цель:** Убрать ручной ввод данных в админке через умные справочники с автопоиском, визуальным выбором флагов, каскадными фильтрами и автоматическим расчётом очков.

**Проблемы которые решаем:**

- ❌ Ручной ввод Country ID → ✅ Выпадающий список с поиском
- ❌ Флаги как текст → ✅ Визуальный превью
- ❌ Ручной расчёт очков → ✅ Автоматический (база × множитель)
- ❌ Нет связи менеджер-достижение → ✅ Связь через FK с каскадной фильтрацией
- ❌ Одинаковые достижения вручную → ✅ Массовое создание
- ❌ Данные рассинхронизированы → ✅ Справочники связаны через FK

## FR-01: Менеджеры

### Страна — выпадающий список с поиском

**Что:** При создании/редактировании менеджера страна выбирается через Searchable Dropdown (Select2 AJAX).

**Как работает:**

1. Пользователь вводит 2+ символа → запрос к `/admin/api/countries?q=...`
2. Выпадающий список показывает: `🏳️ Belarus (BLR)`
3. При выборе страны автоматически заполняются:
   - **Country Code** — только для чтения (например `BLR`)
   - **Flag Preview** — картинка флага 48×48px

**Поля формы менеджера:**

| Поле           | Тип                    | Редактируемое | Автозаполнение        |
| -------------- | ---------------------- | ------------- | --------------------- |
| `name`         | Текст (2-100 символов) | ✅ Да         | —                     |
| `country_id`   | Select2 AJAX Dropdown  | ✅ Да         | —                     |
| `country_code` | Текст (readonly)       | ❌ Нет        | Из `country.code`     |
| `flag_preview` | Картинка 48×48         | ❌ Нет        | Из `country.flag_url` |
| `is_active`    | Чекбокс                | ✅ Да         | `true` по умолчанию   |

**Валидация имени:**

- Минимум 2 символа, максимум 100
- Разрешены: буквы, цифры, пробелы, запятые, точки, апострофы, дефисы
- Паттерн: `^[a-zA-Z0-9\s,.'-]+$`

**Тандемы:**

- Если в имени есть запятая (например `Vlad, whiplash 92`) → показать предупреждение:
  > ⚠️ Обнаружен тандем. Убедитесь что оба игрока представляют одну страну.
- Предупреждение показывается сразу при вводе (JS) и после сохранения (flash message)

## FR-02: Справочник стран

### Управление странами с флагами

**Поля формы страны:**

| Поле        | Тип                    | Редактируемое | Описание                      |
| ----------- | ---------------------- | ------------- | ----------------------------- |
| `code`      | Текст (2-3 символа)    | ✅ Да         | Код страны, например `BLR`    |
| `name`      | Текст (2-100 символов) | ✅ Да         | Название, например `Belarus`  |
| `flag_path` | Выбор файла            | ✅ Да         | Путь к локальному файлу флага |
| `is_active` | Чекбокс                | ✅ Да         | Активна ли страна             |

**Валидация кода:**

- 2-3 заглавные буквы (A-Z)
- Паттерн: `^[A-Z]{2,3}$`
- Автоматическое преобразование в верхний регистр
- Уникальный (нельзя дублировать)

**Флаги:**

- Хранятся в `static/img/flags/`
- Форматы: PNG, SVG, JPG, JPEG
- Максимальный размер: 500 KB
- Превью при выборе файла

> **Примечание:** FlagCDN API (`https://flagcdn.com/w320/{code}.png`) используется как fallback для стран без локального флага. Отдельное поле `flag_url` и переключатель `flag_source_type` не требуются — упрощено по согласованию с заказчиком.

## FR-03: Справочник достижений (Achievement Types)

### Управление типами достижений с калькулятором очков

**Концепция:** Страница "Достижения" — это справочник уникальных типов (TOP1, TOP2, BEST, R3, R1). На ней НЕ создаются достижения для менеджеров. Здесь определяется базовая логика расчёта очков.

**Калькулятор очков в реальном времени:**

При создании/редактировании типа достижения:

1. Выбираешь **Лигу** (1, 2, 2.1, 2.2...) → сразу видишь `base_points` для этой лиги
2. Выбираешь **Сезон** (24/25, 25/26, 26/27...) → сразу видишь `multiplier`
3. **Итоговые очки** пересчитываются на лету: `base_points × multiplier = final_points`

**Пример:**

- Тип: TOP1
- Лига: 2 → `base_points = 300` (из `base_points_l2`)
- Сезон: 25/26 → `multiplier = 1.00`
- **Итого:** `300 × 1.00 = 300.00` очков

**Поля формы типа достижения:**

| Поле             | Тип                | Редактируемое | Описание                      |
| ---------------- | ------------------ | ------------- | ----------------------------- |
| `code`           | Текст (уникальный) | ✅ Да         | Код: TOP1, TOP2, BEST, R3, R1 |
| `name`           | Текст              | ✅ Да         | Название для отображения      |
| `base_points_l1` | Число              | ✅ Да         | Базовые очки для Лиги 1       |
| `base_points_l2` | Число              | ✅ Да         | Базовые очки для Лиги 2       |
| `is_active`      | Чекбокс            | ✅ Да         | Активен ли тип                |

**Калькулятор (только для просмотра):**

| Поле           | Как считается                                                    |
| -------------- | ---------------------------------------------------------------- |
| `base_points`  | Если выбрана Лига 1 → `base_points_l1`, иначе → `base_points_l2` |
| `multiplier`   | Из `Season.multiplier` при выборе сезона                         |
| `final_points` | `base_points × multiplier` — обновляется в реальном времени      |

> **Примечание:** Привязка достижений к менеджерам происходит НЕ здесь, а в карточке менеджера (см. FR-03.1 ниже).

---

### FR-03.1: Привязка достижений к менеджеру

**Где:** В карточке редактирования менеджера (`/admin/manager/{id}/edit/`)

**Что видит пользователь:**

1. Основная форма менеджера (имя, страна, флаг)
2. Ниже — **Таблица текущих достижений** менеджера:

| Тип       | Лига | Сезон | Очки        | Действие   |
| --------- | ---- | ----- | ----------- | ---------- |
| TOP1      | 1    | 24/25 | 800.00      | ❌ Удалить |
| TOP2      | 2    | 25/26 | 200.00      | ❌ Удалить |
| **Итого** |      |       | **1000.00** |            |

3. Кнопка **"➕ Добавить достижение"** — открывает модальное окно или инлайн-форму

**Добавление достижения менеджеру:**

Форма добавления:

- **Тип достижения** (Select2 AJAX) — выбираешь из справочника
- **Лига** (Select2 AJAX)
- **Сезон** (Select2 AJAX, фильтруется по лиге — каскад)
- **Очки** — считаются автоматически (readonly)

Кнопка "Добавить ещё" — можно добавить несколько достижений за раз без перезагрузки страницы.

Кнопка "Сохранить все" — сохраняет все добавленные достижения и пересчитывает рейтинг.

**Валидация:**

- Нельзя добавить дубликат `(type_id + league_id + season_id)` для одного менеджера
- При попытке дубликата → сообщение: _"У этого менеджера уже есть TOP1 в Лиге 1 за сезон 25/26"_
- Каскадная фильтрация: Лига 2.1/2.2 → только сезоны с 2025 года

## FR-04: Массовое добавление достижений менеджеру

### Добавление нескольких достижений одному менеджеру за раз

**Где:** В карточке редактирования менеджера (`/admin/manager/{id}/edit/`)

**Как работает:**

1. Пользователь нажимает **"➕ Добавить достижение"**
2. Открывается форма добавления (модальное окно или инлайн)
3. Заполняет: Тип + Лига + Сезон → видит расчёт очков
4. Нажимает **"Добавить ещё"** — строка добавляется в таблицу, форма очищается для следующей записи
5. Повторяет столько раз сколько нужно
6. Нажимает **"Сохранить все"** — все достижения сохраняются, рейтинг пересчитывается

**Таблица достижений менеджера:**

| Тип       | Лига | Сезон | Базовые очки | Множитель | Итого       | Действие |
| --------- | ---- | ----- | ------------ | --------- | ----------- | -------- |
| TOP1      | 1    | 24/25 | 800          | ×1.00     | 800.00      | ❌       |
| TOP2      | 2    | 25/26 | 200          | ×1.00     | 200.00      | ❌       |
| BEST      | 1    | 24/25 | 50           | ×1.00     | 50.00       | ❌       |
| **Итого** |      |       |              |           | **1050.00** |          |

**Валидация при добавлении:**

- Нельзя добавить дубликат `(type_id + league_id + season_id)` → подсветка строки красным
- Сезон должен быть совместим с лигой → подсветка строки жёлтым
- Все поля обязательны → кнопка "Сохранить все" неактивна пока есть ошибки

**В колонке списка менеджеров:**

- Показывается количество достижений: `Управление наградами (N)`
- Клик → переход к редактированию менеджера с таблицей достижений

## FR-05: Авторасчёт рейтинга

### Триггеры пересчёта

Рейтинг пересчитывается автоматически при:

| Событие                       | Приоритет | Что пересчитывается         |
| ----------------------------- | --------- | --------------------------- |
| Достижение создано            | Высокий   | Рейтинг менеджера           |
| Достижение изменено           | Высокий   | Рейтинг менеджера           |
| Достижение удалено            | Высокий   | Рейтинг менеджера           |
| Множитель сезона изменён      | Средний   | Все достижения этого сезона |
| Очки типа достижения изменены | Средний   | Все достижения этого типа   |

**Режим:** Синхронный (для MVP, < 1000 достижений)

**Инвалидация кэша:**

- Кэш leaderboard (TTL 5 минут) сбрасывается при любом изменении данных
- При следующем запросе к главной странице — рейтинг пересчитывается заново

## API эндпоинты

### API-001: GET `/admin/api/countries`

**Назначение:** Поиск стран для Select2 dropdown

**Параметры:**
| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `q` | string | ❌ | Поисковый запрос (имя или код страны) |
| `page` | int | ❌ | Номер страницы (по умолчанию 1) |
| `page_size` | int | ❌ | Размер страницы (по умолчанию 20, макс 100) |

**Ответ:**

```json
{
  "items": [
    {
      "id": 1,
      "code": "BLR",
      "name": "Belarus",

      "is_active": true
    }
  ],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 28,
    "has_more": true
  }
}
```

## Валидация

### VR-001: Имя менеджера

- Минимум 2, максимум 100 символов
- Паттерн: `^[a-zA-Z0-9\s,.'-]+$`
- Не может быть пустым

### VR-002: Код страны

- 2-3 заглавные буквы
- Паттерн: `^[A-Z]{2,3}$`
- Уникальный
- Автоматическое преобразование в верхний регистр

### VR-003: Уникальность достижений

- Комбинация `(manager_id, type_id, league_id, season_id)` уникальна
- При дубликате → сообщение с ссылкой на редактирование существующего

### VR-004: Совместимость лиги и сезона

- Лига `2.1` и `2.2` доступна только с сезона `25/26` (`start_year >= 2025`)
- Лига `1` и `2` доступны для всех сезонов

### VR-005: Расчёт очков

- Формула: `final_points = base_points × multiplier`
- Округление: 2 знака после запятой
- Очки не могут быть отрицательными (`min: 0`)
- Поля `base_points`, `multiplier`, `final_points` — только для чтения

- Задержка поиска: 250ms

- Градиентный фон: `linear-gradient(135deg, #667eea, #764ba2)`
- Белый текст
- Padding: 15px 20px
- Иконка: 🏆
- Размер значения: 24px bold

### Прогресс-бар (bulk операции)

- Высота: 20px
- Заполнение: `linear-gradient(90deg, #4CAF50, #8BC34A)`
