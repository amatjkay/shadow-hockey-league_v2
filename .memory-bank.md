## 🎯 Ключевые решения

### Архитектура

- **Application Factory** pattern
- **SQLAlchemy** ORM с SQLite
- **Flask-Caching** с Redis (fallback на SimpleCache)
- **Prometheus метрики** (только экспорт, без сервера)

### VPS (Ubuntu 22.04)

- **Хостинг:** ztv.su
- **Ресурсы:** 1 CPU, 1GB RAM, 10GB диск
- **Веб-сервер:** Nginx + Gunicorn (4 workers)
- **БД:** SQLite (dev.db)
- **Redis:** 128MB лимит памяти

### Мониторинг

- **VPS мониторинг:** ztv.su панель (CPU/RAM/диск/трафик)
- **Health check:** `/health` endpoint
- **Метрики:** `/metrics` (Prometheus format, без сервера)
- **Алерты:** отложено

### Запреты/Ограничения

- ❌ **Prometheus сервер** — не устанавливается (экономия RAM)
- ❌ **Grafana** — не используется (экономия RAM)
- ❌ **Uptime Kuma** — отклонено (экономия RAM)
- ❌ **PostgreSQL миграция** — только документация (без реализации)

## 🔧 Конфигурация

### Переменные окружения (.env)

```bash
FLASK_ENV=production
DATABASE_URL=sqlite:///dev.db
SECRET_KEY=xxx
LOG_LEVEL=INFO
ENABLE_API=false

# Redis (Этап 1)
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0
```

### Redis настройка (/etc/redis/redis.conf)

```
maxmemory 128mb
maxmemory-policy allkeys-lru
```
